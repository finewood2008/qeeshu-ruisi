# 企数睿思 (QEESHU RUISI) 咨询超级助理 - 产品需求文档 (PRD)

## 1. 产品概述 (Product Overview)
### 1.1 产品愿景与定位
**企数睿思 (QEESHU RUISI)** 是专为**咨询行业**（如管理咨询、战略规划、数字化转型等）打造的软硬一体化“超级大脑”。
它旨在通过私有化边缘计算硬件 (Qee-Box) 结合强大的 RAG (检索增强生成) 引擎，将咨询公司积累数十年的历史交付文档（PDF 报告、PPT 方案、高管访谈录音）等非结构化“死”资产，转化为即时响应、高度精准的**生产力辅助工具**。

**核心定位**：非通用型 AI，而是深度理解咨询逻辑、方法论体系和商业报告标准的**专属智力外挂**。

### 1.2 目标用户与使用场景
*   **初/中级咨询顾问**：通过系统快速检索历史相似案例，并利用 AI 撰写助手弥补经验不足，提高报告交付质量。
*   **高级顾问/项目经理**：利用 Smart CRM 监控所负责客户群的经营健康度，发现商机；管理和配置机构的专属知识产权（IP/方法论）。
*   **知识管理专员 (KM)**：负责日常物理资产的上传、向量化管理和 Qee-Box 边缘节点的算力监控。

## 2. 核心功能需求详述 (Core Features Detail)
本产品前端 GUI 基于 React 构建，整体架构划分为 6 个核心交互板块。

### 2.1 智能工作台 (Dashboard)
*   **功能描述**：顾问日常登入的全局控制塔。
*   **关键要素**：
    *   **核心指标 (Status Cards)**：展示近期检索量、待处理任务数、健康度预警客户数以及边缘算力(CPU/NPU)的实时状态。支持微缩趋势图 (Sparkline)。
    *   **活跃项目辅助 (Active Projects)**：列表展示当前进行中的项目进度。AI 根据当前项目语境，主动推送提示（例如：“检测到缺失供应链风险分析，建议补充”），分为 Warning 和 Success 两种提示级别。
    *   **智能洞察推荐 (AI Feed)**：位于右侧的科技感面板，基于顾问近期的搜索和浏览历史，主动推送内部高匹配度报告资产，引导阅读。

### 2.2 知识库诊断检索 (Search)
*   **功能描述**：咨询资产的核心检索引擎。采用左侧列表 + 右侧洞察面板的结构。
*   **关键要素**：
    *   **多模态结果列表**：支持展示文档 (PDF/PPT) 和音频 (Audio) 检索结果。高亮命中关键词，并通过“匹配度 (Match Score)”进行排序。
    *   **方法论洞察面板 (AI Insight Panel)**：点击任意文档，右侧自动提取该文档的“核心观点”；系统自动识别并建议提取报告中的关键图表。
    *   **追问企数睿思 (AI Chat)**：在右侧面板底部内嵌的对话流，用户可基于选中的历史文档进行深入的 QA（如“总结这篇报告中的人员优化方案”），支持上下文关联的打字机交互动画。

### 2.3 AI 撰写助手 (AI Writer)
*   **功能描述**：全应用最核心的生产力工具。基于块 (Block-based) 的富文本编辑器。
*   **关键要素**：
    *   **所见即所得编辑器**：支持标准文本、多级标题、列表编辑。
    *   **内联 AI 指令 (Slash Command)**：通过在段落首输入 `/` 唤出交互菜单，执行“自动续写”、“插入标准模型”等动作。
    *   **数据转图表**：支持顾问输入原生数据，AI 自动清洗并渲染为交互式前端图表（如使用 Recharts 实现的面积图/柱状图）。
    *   **方法论完整性审查**：右侧面板实时监控左侧正文，检测是否缺失关键逻辑闭环（例如：漏掉了四象限评估模型）。若检测到缺失，提供**【一键让 AI 补充模型分析框架】**按钮，自动在左侧插入并渲染模型区块及配图。

### 2.4 融合客户管理 (Smart CRM)
*   **功能描述**：以“经营健康度”为核心维度的轻量级商机管理系统。
*   **关键要素**：
    *   **健康度评分监控**：根据业务互动和爬取的公开数据，生成客户的健康度分数（如：92分 优质；65分 预警）。
    *   **深度诊断策略 (AI Diagnosis)**：对于低分预警客户，企数睿思会主动生成预警说明（如“近期财报利润率连续下滑”），并建议主动推介具体的诊断方案。
    *   **客户档案穿透**：点击客户可查看详细的时间轴（历史与当前交付项目）和该客户专属的知识资产库。

### 2.5 本地资产与算力管理 (Assets)
*   **功能描述**：Qee-Box 边缘计算硬件的虚拟映射面板。
*   **关键要素**：
    *   **算力仪表盘**：动态可视化展示当前 CPU、NPU（AI 处理单元）、内存及存储的使用率。
    *   **向量化处理队列**：展示新上传文档的“多模态解析”进度。当文件处于 `processing` 状态时，应能看到进度条增长，且此时 NPU 的利用率应同步产生峰值波动联动。

### 2.6 系统与方法论设置 (Settings)
*   **功能描述**：管理 AI 底层思考逻辑及咨询公司核心数字资产 (IP) 的配置中心。
*   **关键要素**：
    *   **思考策略模式**：切换“混合模式 / 严格私有 / 通用公开”以决定底层 RAG 的检索优先级。
    *   **预置经典框架**：内置麦肯锡 7S、PESTEL、SWOT 等全球框架。可一键开启。点击模型可查看可视化结构与“AI 调用场景说明”的浮层 (Modal)。
    *   **机构自有框架导入**：高亮显示企业私有的知识产权框架，强制最高调用优先级。
    *   **个人/安全偏好**：设置脱敏规则、音频加密销毁规则及 PII（个人身份信息）移除等。

## 3. 前后端交互打通要求 (Backend SDK Integration Specs)

为实现上述高保真 GUI，需要与底层的 **QEECLAW SDK** 进行深度对接。以下是针对后台 API 开发的核心要求：

### 3.1 QEECLAW 知识库检索与生成 API (RAG Core)
*   **需求描述**：支持前端 Search 板块与 AI Writer 板块的智能生成。
*   **关键接口参数**：
    *   **语义检索 (`/api/rag/search`)**：需要接收 `query`，并返回带匹配度 (Score)、高亮片段 (Highlight) 及源文件类型的结构化对象数组。
    *   **单文档深度 QA (`/api/rag/document-chat`)**：传入 `documentId` 和 `userMessage`，返回基于该特定长文本的流式回答 (SSE Stream)，以支撑前端打字机效果。
    *   **方法论审查 (`/api/ai/methodology-check`)**：实时接收 AI Writer 左侧的长文本块，返回结构化的“合规得分”及缺失项标识。
    *   **结构化生成 (`/api/ai/generate-block`)**：当用户点击“一键补充框架”时，需要不仅返回文本，还能返回诸如 `[chart_data]` 或 `[image_url]` 的特殊 Tag，前端将根据 Tag 渲染相应的 React 组件。

### 3.2 QEECLAW 边缘硬件监控 API (Edge Telemetry)
*   **需求描述**：支撑 Assets 硬件管理看板的动态显示。
*   **关键接口参数**：
    *   **硬件遥测 (`/api/edge/telemetry`)**：建议使用 WebSocket (WS) 或短轮询，实时下发当前 Node 节点的 CPU、NPU、RAM 利用率数据（每 2 秒一次）。
    *   **队列状态流 (`/api/edge/queue-status`)**：对于正在进行的“多模态解析（特别是音频转录和 PDF OCR）”，需返回 `processing_progress` (0-100)，以便前端渲染真实进度条。

### 3.3 Smart CRM 动态健康度服务
*   **需求描述**：后端需要聚合数据进行健康度评估。
*   **关键接口参数**：
    *   **健康度评估引擎 (`/api/crm/health-score`)**：根据客户的历史项目交互频次和公网抓取信息，周期性计算健康度，并在触发阈值（如 < 70分）时主动生成预警 `aiDiagnosis` 文案。

### 3.4 权限与数据隔离安全要求
*   **物理隔离模式控制**：当 Settings 中开启了“物理隔离”开关时，QEECLAW SDK 需拦截所有外网大模型（如 ChatGPT/Claude）的 API 请求，强制 fallback 到本地部署的离线开源模型。
*   **动态脱敏 (PII)**：文档上传接口 `/api/assets/upload` 需要支持 `autoMasking` 参数，在落盘向量化之前即完成对人名、财务数字的哈希脱敏处理。
