# 企数睿思功能可用性审计与开发清单

更新时间：2026-04-09

适用范围：

- `qeeshu-ruisi` 桌面版
- Electron 本地优先模式
- 当前代码基线下的前端交互与本地服务能力

## 1. 审计目的

本文件用于回答三个问题：

1. 当前页面上哪些功能已经可用
2. 哪些入口“看起来能点”，但实际上没有接业务动作
3. 哪些功能仍停留在原型期页面，还没有接真实业务链路

这里的“可用性”按三类划分：

- `已可用`：有真实事件处理，且已经接到本地数据或平台数据
- `部分可用`：入口可点，但只有局部能力，或只是本地状态变更
- `不可用`：有按钮/入口外观，但没有有效业务动作

## 2. 总体结论

当前版本已经不再是纯静态原型，以下能力已经形成真实闭环：

- 本地运行时接入配置保存与清除
- 本地 Dashboard / Search / Assets / CRM / AI Writer / Methodology 数据加载
- 本地知识追问会话保存
- 本地客户资料与商机保存
- 本地知识资产登记
- 本地写作草稿保存与回读
- 方法论策略与框架开关持久化
- AI Writer 本地执行日志记录
- AI Writer 块级编辑、图表数据源编辑、正文插入与块操作
- AI Writer HTML 专业报告导出
- AI Writer Word 兼容文档导出
- Assets 本地文件选择导入与文件路径回显
- Assets 本地删除资产、重新索引、导出资产摘要
- 方法论本地文件导入、自有框架启停与本地保存
- 方法论本地结构提炼摘要、自有框架详情查看与优先级调整
- Search 高级筛选实际过滤与图表引用到撰写助手
- CRM 资产查看、导出摘要、跳转检索与导入撰写助手

但页面上仍然存在一批“视觉上像成品、交互上未闭环”的入口，主要集中在：

- Dashboard 的跳转与处理按钮
- Search 的高级筛选、图表引用动作
- AI Writer 的富文本工具栏、导出、图表编辑
- CRM 的方案生成、案例跳转、资产查看
- Assets 的设备设置、筛选、资产查看
- System Settings 的个人资料/偏好/安全区保存动作

## 3. 页面级审计结果

### 3.1 应用框架层

文件：

- `src/App.js`

状态：

- `已可用`
  - 左侧主导航切页
  - 方法论、系统设置切页
  - 连接异常/未配置提示卡片中的“前往配置接入”
- `部分可用`
  - 顶部全局搜索已支持把关键词带入检索页，但还不是跨模块统一搜索引擎
  - 顶部通知按钮已可打开本地通知中心，但没有独立通知管理页
  - “新建跟进事项”已可创建并写入本地存储，但还没有完整事项列表页

结论：

- 外壳导航可用
- 顶栏已经形成基础可用闭环，但仍缺少完整的全局入口页

### 3.2 Dashboard

文件：

- `src/views/Dashboard.js`

状态：

- `已可用`
  - 工作台数据加载
  - 卡片、项目、资产、预警展示
  - 设备卡已改成真实“在线状态 / 最近上报”口径，不再展示伪算力感指标
- `部分可用`
  - “查看全部项目”按钮可跳转 CRM，但项目项本身还没有明细页
  - 推荐卡片箭头可跳转 Search，但当前只保留真实推荐条目，不再展示前端推导匹配分
  - “进入 CRM 处理”按钮可跳转 CRM
  - 文档项、预警项本身还没有独立明细页

结论：

- Dashboard 已具备基础页面联动
- 当前重点不再是补跳转，而是为项目项、文档项、预警项补更细粒度的明细页

### 3.3 Search

文件：

- `src/views/Search.js`

状态：

- `已可用`
  - 左侧检索请求
  - 检索结果切换
  - 本地桌面模式下的知识追问
  - 追问历史恢复
  - 高级筛选里的资产类型、时间范围过滤
  - 检索结果加入 AI Writer
- `部分可用`
  - 当前只保留真实资料引用入口；图表自动抽取与缩略图预览仍未开放
- `不可用`
  - 真正的图表自动抽取与缩略图渲染
  - 更细粒度的多条件组合过滤

结论：

- Search 核心检索、本地追问、筛选和写作联动已经形成闭环
- 当前已明确移除前端自动生成的匹配度、推荐追问和图表引用占位动作

### 3.4 AI Writer

文件：

- `src/views/AIWriter.js`

状态：

- `已可用`
  - 草稿标题编辑
  - 本地草稿保存
  - 本地草稿回读
  - 新建草稿
  - Slash Menu / 推荐动作触发
  - 本地 AI 执行日志写入
  - 本地模式下展示真实草稿数和执行记录数
  - 富文本工具栏对选中块的加粗/斜体/对齐/列表切换
  - 内容块选中、复制、删除
  - 图表数据源编辑弹窗
  - 正文块快速插入
  - Markdown 导出
  - HTML 专业报告导出
  - Word 兼容文档导出
- `部分可用`
  - 若本地未连接云端模型，则不会展示额度/用量面板，而是明确显示本地模式说明
- `不可用`
  - 真正的流式生成式编辑器
  - 在未连云情况下的真实 block generate / methodology check 能力
  - Word/PDF 等正式交付格式导出

结论：

- AI Writer 已从“静态演示页”推进到“可用的本地块编辑器”
- 距离正式写作器还差两件事：真正生成接口、正式 PDF/Docx 级交付格式导出

### 3.5 CRM

文件：

- `src/views/CRM.js`

状态：

- `已可用`
  - 客户列表展示
  - 客户详情页切换
  - 本地新增客户
  - 本地编辑客户
  - 本地新增商机

结论：

- 与 Search / Writer 的主要联动动作已经打通

## 4. 不可用功能清单

以下功能建议统一视为“待开发”，不要再按“已上线能力”对外描述：

- Dashboard 全部跳转/处理按钮
- Search 图表自动抽取与缩略图渲染
- AI Writer 智能配图
- AI Writer SSE 流式回传
- Assets 设备设置
- Assets 文件预览
- Methodology 深度文档解析与共享
- System Settings 全局主题系统
- 全局通知、全局任务创建、全局搜索

## 5. 建议开发优先级

### P0：补齐“按钮有外观但无动作”的高误导入口

- Dashboard 的三个主按钮
- Assets 设备设置与查看按钮
- 全局通知、全局任务创建、全局搜索

目标：

- 要么补真实动作
- 要么明确禁用并给出“建设中”提示

### P1：补页面间联动闭环

- Search 结果加入 AI Writer
- CRM 客户跳转到 Search 相似案例
- Dashboard 预警跳转 CRM / Search / Writer

目标：

- 从“单页面原型”升级成“产品流程”

### P2：补真实业务能力

- AI Writer 流式生成、智能配图与正式 PDF/Docx 交付
- Assets 文件预览、真正的索引流水回传
- Methodology 深度文档提炼、共享与编排能力
- 全局通知中心、全局搜索与任务入口

## 6. 推荐实施方式

建议后续开发按以下顺序推进：

1. 先处理所有无动作按钮，避免继续制造“页面可点击但不可用”的错觉
2. 再补页面联动，优先打通 Dashboard -> CRM / Search / Writer
3. 最后补重能力模块：流式生成、文件预览、方法论知识工程与全局能力

## 7. 对外口径建议

当前版本更准确的表述应该是：

> 企数睿思桌面版已经具备本地优先的数据底座和主要业务闭环，新库默认以空态启动，不再注入预置演示业务数据；但仍有部分全局能力和深度产品能力尚未全部完成正式产品化开发。

不建议对外直接表述为：

> 所有页面均已完成开发
